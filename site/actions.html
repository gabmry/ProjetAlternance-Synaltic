4 actions sont disponibles au joueur:

 Coder bien: Code une quantité normale avec
une faible chance d’avoir des bugs

Coder vite: Code une quantité élevée mais
avec une chance élevée d’avoir des bugs

Tester: Convertit du taux de code buggé
en taux de complétion

Abandonner: Les taux de complétion et de
bug du joueur et de l’IA sont remis à 0
